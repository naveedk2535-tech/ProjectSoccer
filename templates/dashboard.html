What's a Value Bet?

A value bet is when our model disagrees with the bookmaker — and we think the bookmaker is wrong.

Example: Our model says Team X has a 35% chance to win.

The bookmaker offers odds of 4.00, which implies only a 25% chance.

That 10% gap is the edge. Over many bets, this edge = profit.

Edge % = How wrong we think the bookmaker is

Kelly % = Suggested stake as % of your total bankroll

Click any value bet to see the full match analysis.

Real example from this dashboard:

Tottenham vs Nottingham Forest

Our model says: Forest wins 45% of the time

Bookmaker says: Forest wins 34% of the time (odds 2.96)

Forest actually wins 45% but you're getting paid as if they only win 34% — you're being overpaid.

Over 100 bets like this at $10 each:

Win ~45 times x $19.60 profit = $882

Lose ~55 times x $10 lost = $550

NET PROFIT: +$332 from $1,000 staked = +33% ROI

You WILL lose individual bets. That's normal. The edge shows up over volume — like how a casino doesn't win every hand but always profits over thousands. You're the casino.

Our backtest proves it: across 348 historical value bets, the model returned +31.3% ROI.

How Winnings Work

Odds tell you what you'll get back if you win:

Odds of 1.85 = Bet $10, get back $18.50 ($8.50 profit)

Odds of 3.00 = Bet $10, get back $30.00 ($20.00 profit)

Odds of 6.80 = Bet $10, get back $68.00 ($58.00 profit)

Higher odds = bigger payout but lower chance of winning. The model helps you find cases where the payout is bigger than the risk justifies.

The Four Models — How They Work

1. Poisson + Dixon-Coles (35% weight)

Calculates each team's Attack Strength (goals scored vs league average) and Defence Weakness (goals conceded vs league average). These produce an expected goals number for each side. Then uses the Poisson distribution to generate a 7x7 scoreline probability matrix (every possible score from 0-0 to 6-6). The Dixon-Coles correction adjusts for low-scoring games (0-0, 1-0, 1-1) which Poisson alone underestimates. Recent matches are weighted more heavily (time decay).

Outputs: H/D/A %, Over 2.5, BTTS, exact scoreline probabilities

2. XGBoost Machine Learning (30% weight)

Gradient-boosted decision tree trained on 1,000+ historical matches using 20 features:

Elo ratings (home & away)

Attack & Defence strength (x4)

Form last 5 games (both teams)

Days rest since last match

Head-to-head win rate

Goals per game this season

Referee avg goals/cards

Month of season

Sentiment scores

Promoted team flag

Streak length

Pinnacle implied probability

Finds non-linear patterns humans miss. Top feature: head-to-head history (33% importance).

3. Elo Rating System (25% weight)

Chess-style rating. Every team starts at 1500. After each match, the winner gains points and loser drops — more points change hands in upsets. Includes home advantage bonus (+50 Elo), goal difference multiplier (bigger wins = bigger rating change), and season decay (ratings regress toward average between seasons so promoted teams aren't stuck at 1500 forever). Tracks home/away Elo separately, plus last 5 and 10 game form, and streak detection.

4. Sentiment Analysis (10% weight)

Pulls posts from Reddit (r/soccer, r/PremierLeague) and news headlines (NewsAPI). Scores text using VADER sentiment analysis (-1 negative to +1 positive). Tracks mention volume (sudden spikes = something happened). Flags keywords: injury, sacked, ban, transfer, etc. Weakest signal by design — it's a tiebreaker, not a primary predictor.

How the Ensemble Works

Each model produces H/D/A probabilities. The ensemble takes a weighted average:

Final = (Poisson x 0.35) + (XGBoost x 0.30) + (Elo x 0.25) + (Sentiment x 0.10)

If a model fails (e.g., XGBoost can't load), the remaining models' weights are re-normalised so they still sum to 100%. The ensemble then compares its probabilities against bookmaker odds to find value.

Kelly Criterion sizes the bet: f = (bp - q) / b, where b = odds-1, p = model probability, q = 1-p. We use quarter-Kelly (x0.25) because full Kelly assumes perfect edge estimation, which we don't have.
